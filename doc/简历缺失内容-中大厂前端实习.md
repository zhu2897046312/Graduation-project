### 1.1 电商购物平台 PC 端（Nuxt / C 端）

**当前问题**：只写完成购物车、结算等流程，无量化、无难点深度。

**建议补充（可直接或改写进简历）**：

| 维度 | 缺失内容 | 补充表述（示例） |
|------|----------|------------------|
| **业务价值** | 支撑多少日活/订单量 | 项目已上线生产环境（如 earring18.com），**支撑完整 C 端购物与支付闭环**；若可提供：可写「支撑 XX 日活 / 月订单量级」或「0 支付相关线上故障」 |
| **首屏/性能** | 首屏加载提升多少 | **通过解决 Nuxt SSR 样式 FOUC**，实现**服务端样式预渲染**，首屏无样式闪烁，**CLS（累积布局偏移）显著改善**，利于 Core Web Vitals；若做过 Lighthouse：可写「首屏 LCP/CLS 优化至 XX」 |
| **核心难点与方案** | FOUC 一笔带过 | **技术方案**：Nuxt 3.12 移除 `ssrContext.styles` 后，使用 **`@css-render/vue3-ssr` 在服务端 collect 样式，通过 `nuxtApp.ssrContext.head.push` 注入 `<head>`**，保证 HTML 直出即带样式。<br>**效果**：消除 FOUC，**首屏白屏/样式闪烁问题解决**，搜索引擎抓取时页面完整，**SEO 与体验双提升** |
| **数据一致性** | 设备指纹只说「解决」 | **技术方案**：**Cookie 持久化设备 ID + 服务端模块级变量缓存**同一请求内 UUID，客户端从 Cookie 读取，避免 SSR 双端执行导致双 UUID。<br>**效果**：**设备指纹服务端/客户端一致性 100%**，游客购物车、风控识别稳定可用 |
| **工程/稳定性** | 请求层降低多少报错率 | 统一封装请求层（Token、设备指纹、错误处理），**接口调用处零散错误处理收敛到拦截器**，未登录/过期统一跳转；可表述为「**统一请求与错误处理，减少重复逻辑，提升可维护性与线上问题定位效率**」 |

**一句话核心成果（可作项目首句）**：

> 负责电商 PC 端前端开发与部分架构设计，**独立排查并解决 Nuxt SSR 下 Naive UI 样式 FOUC 及服务端/客户端设备指纹不一致两大线上问题**，实现**服务端样式预渲染与设备 ID 同构一致**，提升首屏体验与 SEO；完成商品、购物车、结算、PayPal 支付与订单全流程，项目已上线生产。

## 二、技术栈：从「会什么」到「掌握深度 + 中大厂契合度」

### 2.1 技能分层（简历「技能特长」建议顺序）

**第一层：核心框架与语言（大厂必看）**

- **Vue 3**：Composition API、`<script setup>`、**响应式原理（ref/reactive、依赖收集）**、**自定义 Hooks/Composables 设计**（如 useCartHook、useSiteInfo）；生命周期与 SSR 下的使用注意点
- **TypeScript**：**类型体系**（接口、泛型、类型推断）、在接口、Props、Store 中的落地，**减少运行时错误与提升重构安全**
- **前端工程化**：**Vite 配置**（路径别名、环境变量、开发代理、transpile）、与 Vue CLI 的差异（ESM 开发、Rollup 构建、冷启动与 Tree-shaking）；若接触过 Webpack：可写「了解 Webpack 与 Vite 的构建差异与优化思路」

**第二层：工程化与架构能力**

- **请求层设计**：**统一封装**（baseURL、Token、设备指纹/Header）、**响应拦截**（业务 code、登录态失效、错误提示），接口层与业务解耦
- **状态管理**：Pinia（登录态、权限、持久化）/ Nuxt useState（跨页面共享、SSR 友好），**状态边界划分与多页面共享**
- **权限与安全**：**RBAC + 自定义指令**实现按钮级控制；**登录态与多端 Token 来源**（Header/Query/Cookie）兼容

**第三层：性能与体验**

- **性能优化**：**SSR 首屏**（useAsyncData 数据预取、服务端样式注入）、**组件懒加载/按需加载**、图片懒加载；**Core Web Vitals（CLS/LCP）** 的实践（如消除 FOUC）
- **SEO**：SSR/SSG、动态 TDK、useAsyncData 与首屏直出

**第四层：UI 与工具库**

- Naive UI / Ant Design Vue、Tailwind CSS、ECharts、WangEditor、cropperjs 等：写「熟练使用」即可，不必与 Vue/TS 并列前排
